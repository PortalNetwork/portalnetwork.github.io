Portal Network’s contribution to MetaMask

ENS is here!

A few days ago, MetaMask has announced its newest update 4.9.0, which included the integration with the ENS (Ethereum Name Service) resolver. This means that MetaMask can now support loading ENS website (.eth) that are hosted on IPFS. When visiting decentralized sites, users can simply type in the domain name instead of long bytes of hashes. It is definitely a big step towards a better solution for easier access to the decentralized contents.

This has been one of the efforts that Portal Network has been putting for more user adoption in entire blockchain ecosystem.

We are glad that our contribution to MetaMask has been accepted for their newest update!

How to install MetaMask?

Visit MetaMask official website and click “GET CHROME EXTENSION”. It will then guide you to the Chrome Store where you can easily download the MetaMask extension.

Introducing ENS

ENS compatibility is the main feature of this update. This implementation deals with the mapping between ENS and the decentralized resources address in an elegant way. Simply type a “.eth” URL and MetaMask will resolve ENS domains to IPFS hash and redirect the browser to the related content. Hosting sites on ENS and IPFS reduce the risk of hacking and phishing that comes out one after another.
